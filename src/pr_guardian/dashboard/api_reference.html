API Reference

API Reference

REST API at http://localhost:8000/api. All endpoints return JSON.

Agent API

API Key

Endpoints for external agents and CI bots. Authenticate with Authorization: Bearer prg_*. Create keys in Admin.

Authentication

curl -H "Authorization: Bearer prg_your_key_here" https://your-instance/api/agent/reviews

Scopes: read (list/get reviews) | write (dismiss, re-review, trigger review)

GET/api/agent/reviews

List reviews. Scope: read

ParamTypeDescription
repostringFilter by repository (e.g. owner/repo)
pr_idstringFilter by PR number
limitintMax results (default 20)
GET/api/agent/reviews/{review_id}

Get a review with all findings. Scope: read

POST/api/agent/findings/{finding_id}/dismiss

Dismiss a finding. Scope: write

{"status": "false_positive", "comment": "Not applicable in this context"}

Status: by_design | false_positive | acknowledged | will_fix

POST/api/agent/reviews/{review_id}/re-review

Re-evaluate existing findings against current code. Scope: write

POST/api/agent/review

Trigger a full review for a PR. Respects previously dismissed findings. Scope: write

{"pr_url": "https://github.com/owner/repo/pull/123", "post_comment": true}

Example: Agent workflow

# 1. Get latest review for a PR curl -H "Authorization: Bearer $KEY" \ "https://host/api/agent/reviews?repo=owner/repo&pr_id=42&limit=1" # 2. Read findings curl -H "Authorization: Bearer $KEY" \ "https://host/api/agent/reviews/REVIEW_ID" # 3. Dismiss a false positive curl -X POST -H "Authorization: Bearer $KEY" \ -H "Content-Type: application/json" \ -d '{"status":"false_positive","comment":"checked - not reachable"}' \ "https://host/api/agent/findings/FINDING_ID/dismiss" # 4. Request re-review curl -X POST -H "Authorization: Bearer $KEY" \ "https://host/api/agent/reviews/REVIEW_ID/re-review"

Reviews

Entra ID
POST/api/review

Trigger a review by PR URL. Returns immediately; review runs in background.

{"pr_url": "https://github.com/owner/repo/pull/123", "dry_run": false, "post_comment": false}
POST/api/review/repo

Review an entire repository at a given ref as a synthetic PR. Runs the full PR review pipeline over every file in the repo. Small repos only — rejected if max_files is exceeded. Platform side-effects (comments, status checks) are suppressed.

{"repo": "owner/repo", "platform": "github", "ref": "HEAD", "max_files": 300}
GET/api/dashboard/reviews

Paginated list of reviews with optional filters.

ParamTypeDescription
limitintMax results (1-200, default 50)
offsetintPagination offset
repostringFilter by repository
decisionstringauto_approve, human_review, reject, hard_block
authorstringFilter by PR author
GET/api/dashboard/my-reviews?author=octocat

Convenience endpoint for a specific author's reviews.

ParamTypeDescription
authorstringRequired. PR author username
limitintMax results (1-100, default 10)
decisionstringFilter by decision
GET/api/dashboard/reviews/{review_id}

Full review detail with findings enriched with dismissal status.

GET/api/dashboard/active

Currently in-progress reviews.

DELETE/api/dashboard/reviews/{review_id}

Cancel a stuck review.

Finding Dismissals

Entra ID
POST/api/dashboard/findings/{finding_id}/dismiss

Dismiss a finding with a status and optional comment.

{"status": "false_positive", "comment": "Not applicable"}

Status: by_design | false_positive | acknowledged | will_fix

POST/api/dashboard/findings/batch-dismiss

Dismiss multiple findings in one request.

{"finding_ids": ["uuid-1", "uuid-2"], "status": "false_positive", "comment": "Batch dismissed"}

Returns: {"dismissed": 2, "not_found": [], "signatures": [...]}

DELETE/api/dashboard/dismissals/{dismissal_id}

Remove a dismissal (un-dismiss).

Re-Review

Entra ID
POST/api/dashboard/reviews/{review_id}/re-review

Focused re-evaluation of original findings. Fetches incremental diff, asks agents if findings are still valid. Does NOT run a full review or find new issues.

{"status": "queued", "pr_id": "123", "mode": "re_evaluate", "findings_to_evaluate": 8}

Scans

POST/api/scan/recent

Recent changes scan on merged code.

{"repo": "owner/repo", "platform": "github", "time_window_days": 7}
POST/api/scan/maintenance

Maintenance scan for stale files.

{"repo": "owner/repo", "platform": "github", "staleness_months": 6, "max_files": 50}
GET/api/dashboard/scans

Paginated list of scans. Params: limit, offset, repo, scan_type.

GET/api/dashboard/scans/{scan_id}

Full scan detail with findings.

GET/api/dashboard/scan-stats

Aggregate scan statistics.
