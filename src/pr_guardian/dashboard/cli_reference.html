CLI Reference

CLI Reference

All commands available via pr-guardian. The CLI talks directly to the database — no running server required.

Service

serve

Start the PR Guardian HTTP service (webhooks + dashboard).

pr-guardian serve [--host 0.0.0.0] [--port 8000]

Configuration

validate

Validate a review.yml configuration file.

pr-guardian validate [--config review.yml]

detect-languages

Detect languages in changed files. Reads from arguments or stdin.

pr-guardian detect-languages [--diff-target main] [--output result.json] file1.py file2.ts

dry-run

Run triage classification without invoking AI agents. Useful for testing risk rules.

pr-guardian dry-run [--config .] [--diff-target main] file1.py file2.ts

Reviews

reviews

List recent reviews with optional filters.

pr-guardian reviews [--limit 20] [--repo owner/repo] [--decision human_review] [--json-output]
OptionDefaultDescription
--limit20Max reviews to show
--repo—Filter by repository
--decision—auto_approve, human_review, reject, hard_block
--json-output—Output raw JSON

review <review_id>

Show full review detail with findings, agent verdicts, and dismissal status.

pr-guardian review <uuid> [--json-output]

my-reviews <author>

Show recent reviews for a specific PR author.

pr-guardian my-reviews octocat [--limit 10] [--decision human_review] [--json-output]

Finding Management

dismiss <finding_id>

Dismiss a single finding by UUID.

pr-guardian dismiss <uuid> --status false_positive [--comment "reason"]
OptionRequiredValues
--statusYesby_design, false_positive, acknowledged, will_fix
--commentNoFree-text explanation

batch-dismiss <review_id>

Dismiss multiple findings from a review. Supports severity threshold and ID filtering.

pr-guardian batch-dismiss <uuid> --status false_positive [--severity low] [--finding-ids "id1,id2"]
OptionRequiredDescription
--statusYesby_design, false_positive, acknowledged, will_fix
--commentNoFree-text explanation
--finding-idsNoComma-separated UUIDs (default: all)
--severityNoDismiss at this severity or lower (low, medium, high, critical)

re-review <review_id>

Focused re-evaluation of original findings. Fetches incremental diff since last reviewed commit and asks agents if findings are still valid. Does NOT find new issues.

pr-guardian re-review <uuid> [--post-comment / --no-comment]

Scans

scan-recent

Run a recent-changes scan on merged code.

pr-guardian scan-recent --repo owner/repo [--platform github] [--days 7] [--since 2024-01-01]

scan-maintenance

Find stale files needing attention.

pr-guardian scan-maintenance --repo owner/repo [--platform github] [--staleness 6] [--max-files 50]

Agent Workflow Example

A typical agent or developer workflow for triaging review findings:

# 1. Pull your reviews
pr-guardian my-reviews octocat --json-output

# 2. Inspect findings for a specific review
pr-guardian review <review-id>

# 3. Dismiss false positives
pr-guardian dismiss <finding-id> --status false_positive

# 4. Batch dismiss low-severity noise
pr-guardian batch-dismiss <review-id> --status acknowledged --severity low

# 5. Re-review with dismissals applied
pr-guardian re-review <review-id>
